Look at the keys in your keyring

When you generate a PGP key, a keypair is generated. This keypair contains a public key and a private key. The public key can be shared with everyone but the private key should not be disclosed to anyone. The private key is only used by you to decrypt your messages while everyone who wants to email you, encrypts your messages with your public key.

Public keys

List your public keys

    $gpg --list-keys
    pub   4096R/D30EAACD 2013-07-04 [expires: 2014-04-30]
    uid                  Remus Lupin (The Marauders Map Secret Key) <loopylupin@gmail.com>
    sub   4096R/7761B227 2013-07-04 [expires: 2014-04-30]

The above command lists all your public keypairs. Each keypair consists of a master key and a subkey. The master key is identified by pub while the subkey is identified by sub.

Each key also has a key ID. In the above case, the master key has the key ID D30EAACD and the subkey has the key ID 7761B227.

Private keys

List your private keys

    $gpg --list-secret-keys
    sec   4096R/D30EAACD 2013-07-04 [expires: 2014-04-30]
uid                  Remus Lupin (The Marauders Map Secret Key) <loopylupin@gmail.com>
ssb   4096R/7761B227 2013-07-04

Note that the private key and the public key share their key ID but the secret key is identified by sec instead of pub.
