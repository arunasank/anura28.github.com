Generate your cryptography keys

Use the GPG command line interface to create your PGP keys.

  1. Open the Terminal.

  2.         $ gpg2 --gen-key
        gpg (GnuPG) 2.0.20; Copyright (C) 2013 Free Software Foundation, Inc.
        This is free software: you are free to change and redistribute it.
        There is NO WARRANTY, to the extent permitted by law.

        Please select what kind of key you want:
        (1) RSA and RSA (default)
        (2) DSA and Elgamal
        (3) DSA (sign only)
        (4) RSA (sign only)
        Your selection?1

    You can choose any of the four choices.

            RSA keys may be between 1024 and 4096 bits long.
        What keysize do you want? (2048)4096

    Any key size between 2048 and 4096 bits is very secure. If you just press Enter, the default key size(2048 bits) will be chosen)

    	
        Requested keysize is 4096 bits
        Please specify how long the key should be valid.
        0 = key does not expire
        <n>  = key expires in n days
        <n>w = key expires in n weeks
        <n>m = key expires in n months
        <n>y = key expires in n years
        Key is valid for? (0)10m

    If you just press Enter, the key will never expire as that is the default choice.

            Key expires at Wednesday 30 April 2014 02:56:04 PM IST
Is this correct? (y/N)y
        You need a user ID to identify your key; the software constructs the user ID
from the Real Name, Comment and Email Address in this form:
    "Heinrich Heine (Der Dichter) <heinrichh@duesseldorf.de>"

    Real name:Remus Lupin
        E-mail address:loopylupin@gmail.com
        Comment:The Marauders Map Secret Key
        You selected this USER-ID:
    "Remus Lupin (The Marauders Map Secret Key) <loopylupin@gmail.com>"

Change (N)ame, (C)omment, (E)-mail or (O)kay/(Q)uit?O

    It's very important to give authentic details here. Remember that people identify your key using your user ID.

            You need a Passphrase to protect your secret key.*****************

    A passphrase is a long sequence of words with special characters. Your passphrase must be at least 20-30 characters long though there is no limit to the length of your passphrase.

            We need to generate a lot of random bytes. It is a good idea to perform
some other action (type on the keyboard, move the mouse, utilise the
disks) during the prime generation; this gives the random number
generator a better chance to gain enough entropy.

    Perform any actions during this time, like browsing your favourite website, or composing an email. andom activities using the keyboards and the mouse generate your key. A 4096 bit key needs more entropy than a 2048 bit key.
